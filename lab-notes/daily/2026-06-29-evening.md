## 2026-06-29 (Evening) — Arm IK Fix + CLI Flags + MPC + Live Plot
**No new commits** (all changes in working tree)

**What changed:**

### Arm Control Fix
The arm was **not actually moving up** — the logging was reading `sim.arm.get_state()` which returns `self.state`, only updated by `sim.arm.step()`. Since the demo used `sim.engine.set_arm_ctrl()` directly (bypassing `sim.arm.step()`), the logged state was **stale** — always showing z=0.141.

**Root cause chain:**
1. `lekiwi_sim.py` `reset()` set free joint quaternion to `[0,0,0,0]` (invalid) → garbage EE positions
2. Fallback FK model in `armrobot.py` was completely disconnected from MuJoCo's mesh — FK at q=0 gave z=0.031m, MuJoCo's real mesh gave z=0.141m
3. `sim.arm.get_state()` returned stale data because `set_arm_ctrl()` doesn't update `self.state`

**Fixes:**
- `lekiwi_sim.py`: `reset()` now sets free joint quaternion to `[1,0,0,0]` (identity)
- `capture_gif.py`: `true_ee` reads directly from `sim.engine.data.xpos[ee_body_id]` instead of `sim.arm.get_state()`
- `capture_gif.py`: Arm control uses **iterative damped least-squares IK** on MuJoCo's real Jacobian (converges in ~8 iterations for 0.16m error)
- IK restores original qpos after computation so position servos have error to track
- Removed all broken fallback FK/IK code from `capture_gif.py`

**Result:** Arm now tracks with ~0.04m mean error. Reaches z=0.287 (target 0.300) and z=0.044 (target 0.050).

### CLI Flags Added
- `--controller lqr|mpc` — choose base controller (LQR default, MPC with horizon=15 + velocity constraints)
- `--trajectory straight|triangle` — choose base trajectory (straight line or triangle: forward → diagonal → back to origin)
- `--gif` — headless GIF capture
- `--fast` — fewer frames for faster rendering

### MPC Integration
- MPC_LTI from `mpc_lti.py` wired into demo with same Q/R matrices as LQR
- Velocity constraints: vx, vy ∈ [-0.5, 0.5], ω ∈ [-1.0, 1.0]
- Plot title dynamically shows "LQR Control Effort" or "MPC Control Effort"

### Live Plot Improvements
- 3×2 layout: base tracking (ref/noisy/est/true), base innovation, arm tracking, arm innovation, base control effort, arm position error
- Sensor noise + Luenberger observer (base) + EMA estimator (arm)
- Colored waypoint markers in 3D scene
- Dark theme

**Working:** Both controllers (LQR/MPC), both trajectories (straight/triangle), arm tracks EE targets with ~0.04m error, live viewer + growing plot, GIF capture.
**Not working:** Arm position servos (kp=50) are slow to converge for large displacements. No combined state-space. No real pick-and-place (no object in scene).
