## 2026-06-16 — Controllers + Plant Classes
**Last commit:** `6c30dab` — "refactor: optimize kinematics caching and rename mpc solve method to compute"

**Files added:**
- `holonomicmobilerobot.py` — `HolonomicMobileRobot` class (replaces the standalone function)
- `mpc_lti.py` — `MPC_LTI` class with OSQP QP solver
- `pid.py` — `PIDCcontroller` class (buggy — missing `_integral` and `_prev_error` initialization)

**Decisions:**
- **OOP for the robot base.** `HolonomicMobileRobot` with `step()`, `get_state()`, `get_model()`, `set_pose()`. State is `[x, y, theta]`. `step()` takes world-frame velocity, rotates to body frame, computes wheel speeds, integrates state.
- **MPC_LTI with OSQP.** Full LTI MPC with precomputed dynamics matrices (T_bar, S_bar) and cost matrices (H, F). Uses `osqp.OSQP()` for solving the QP. Terminal cost and constraints.
- **PID controller.** Basic PID with P, I, D terms and output limits. Implementation is incomplete.
- **`get_model_matrices()` returns A=I, B=dt\*I** — base modeled as simple discrete integrator.

**Working:** Base kinematics and state integration. MPC_LTI matrix construction and OSQP solve. PID skeleton.
**Not working:** PID has uninitialized `_integral`/`_prev_error`, incomplete output clamping. No arm code. No abstract base classes.
