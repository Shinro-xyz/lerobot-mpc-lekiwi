## 2026-06-29 — MuJoCo Integration + Demos + Tests
**Last commit:** `6f3a094` — "feat: implement pick-and-place demo with LQR base control and arm trajectory planning"

**Files added:**
- `lekiwi_sim.py` — `MuJoCoEngine` and `LeKiwiSim` classes for Python-based MuJoCo simulation
- `capture_demo.py` — pick-and-place demo with 8-phase scripting
- `capture_gif.py` — GIF capture demo with LQR base control + cubic trajectory planning
- `test_pick_and_place.py` — 6 real tests for the pick-and-place pipeline
- `lekiwi_demo.gif` — output GIF
- `.python-version` — Python 3.12.3
- `.zed/settings.json` — Zed editor config

**Decisions:**
- **MuJoCoEngine as a physics backend, NOT a Plant.** The engine is a low-level MuJoCo wrapper. `ArmRobot` and `HolonomicMobileRobot` each hold a reference to the engine and read/write their respective slices of the shared state. Controllers talk to the robot classes, never to MuJoCo directly.
- **LeKiwiSim convenience wrapper.** Creates a shared `MuJoCoEngine` and instantiates `ArmRobot` + `HolonomicMobileRobot` with it. Provides `reset()`, `step()`, `get_state()`.
- **LQR for base position tracking.** Q=diag(100,100,50), R=diag(0.1,0.1,0.1).
- **Cubic trajectory for arm.** Simple `Trajectory` class (not the ABC) with precomputed keyframes.
- **EGL headless rendering.** Both demos use `MUJOCO_GL=egl` for headless GIF capture.
- **Comprehensive test suite.** 6 tests covering base driving, diagonal movement, arm reaching, gripper open/close, full pick-and-place sequence, and reset.

**Working:** Full end-to-end pick-and-place demo with MuJoCo physics, LQR base control, arm trajectory planning, GIF capture. All 6 tests pass.
**Not working:** Base is welded to world in MuJoCo (no free joint) — base pose is always `[0,0,0]` in MuJoCo, with kinematic state tracked separately in `HolonomicMobileRobot.state`. Wheel visual roll is manually updated via `_target_wheel_delta`. No combined state-space (base + arm coupling). No Shinro IDE integration.
