## 2026-06-14 — Base Kinematics + MuJoCo Simulation
**Last commit:** `b7a13d3` — "feat: implement inverse kinematics matrix for lekiwi robot"

**Files:**
- `lekiwi-robot-base.py` — standalone function `MobileRobotBaseKinematics()` computing the holonomic kinematics matrix and its pseudoinverse
- `lekiwi-sim/` — full MuJoCo simulation directory with `mjcf_lcmm_robot.xml`, `so_arm100.xml`, 40+ STL mesh files, and a README

**Decisions:**
- Kinematics was a **standalone function**, not a class. No OOP, no plant abstraction.
- MuJoCo model imported from Fusion 360 via ACDC4Robot plugin. All meshes simplified.
- 3 omni-wheel holonomic base (120° spacing) + SO-ARM100 6-DOF arm + gripper.
- Arm joints: Rotation, Pitch, Elbow, Wrist_Pitch, Wrist_Roll, Jaw — all position-controlled.

**Working:** MuJoCo model loads and renders. Base kinematics function computes the wheel-to-velocity matrix.
**Not working:** No controllers, no plant abstraction, no arm kinematics code.
