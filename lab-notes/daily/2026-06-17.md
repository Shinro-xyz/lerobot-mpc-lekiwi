## 2026-06-17 — Abstract Base Classes + Arm + LQR
**Last commit:** `93333c7` — "feat: extend armrobot with joint limits, offsets, and basic kinematics"

**Files added:**
- `components.py` — Abstract base classes: `Controller`, `Plant`, `StateEstimator`
- `armrobot.py` — `ArmRobot(Plant)` class with joint limits, offsets, and `_homogenous_transform()` stub
- `lqr.py` — `LQR(Controller)` class using `scipy.linalg.solve_discrete_are`

**Decisions:**
- **Three abstract base classes:** `Controller` (compute, reset), `Plant` (get_state, get_model, step), `StateEstimator` (estimate, reset). This is the core architectural pattern that persists throughout the project.
- **ArmRobot as a Plant.** Extends the Plant ABC. Has `num_dof`, `dt`, `state`, `joint_limits`, `joint_offsets`. `get_model()` returns A=I, B=dt\*I.
- **LQR via DARE.** Uses the discrete algebraic Riccati equation solver for optimal gain computation.
- **Arm kinematics stub.** `_homogenous_transform()` started but only handles 'x' axis.

**Working:** ABCs in components.py. LQR controller with DARE solve. ArmRobot skeleton with state tracking.
**Not working:** Arm forward kinematics incomplete. No Jacobian, no inverse kinematics. PID still buggy. No state estimators.
